## Overview

Hiro's AI Assistant brings conversational intelligence to your CRM. Ask questions in natural language and get instant insights about reservations, customers, revenue, and operations—without navigating through menus or building reports.

<Note>
**Powered by:** OpenAI GPT-4 with hospitality-specific training and your real-time operational data.
</Note>

## How It Works

### Query Router

When you ask a question, Hiro's AI follows this process:

<Steps>
  <Step title="Intent Classification">
    Analyzes your question to determine intent:
    - Reservations query?
    - Customer lookup?
    - Revenue analysis?
    - Operational help?
  </Step>
  <Step title="Context Loading">
    Fetches relevant real-time data:
    - Today's reservations across all locations
    - Weekly revenue trends
    - VIP customer status
    - Inactive customer lists
    - General statistics
  </Step>
  <Step title="Response Generation">
    Two modes depending on setup:
    - **OpenAI Mode**: Full GPT-4 response with conversational tone
    - **Structured Mode**: Pre-formatted responses with data tables
  </Step>
  <Step title="Formatting & Delivery">
    Response formatted with:
    - Markdown formatting
    - Emojis for visual clarity
    - Tables and lists
    - Action recommendations
  </Step>
</Steps>

## OpenAI Configuration

### Setup Requirements

To enable full AI Assistant capabilities:

<Steps>
  <Step title="Navigate to Settings">
    Go to Settings → Integrations → OpenAI
  </Step>
  <Step title="Enter API Key">
    Input your OpenAI API key:
    - Get key from https://platform.openai.com
    - Requires paid OpenAI account
    - Recommended: Create dedicated key for Hiro
  </Step>
  <Step title="Configure Model">
    Choose model:
    - **gpt-4-turbo** (Recommended): Best quality, slower
    - **gpt-4**: High quality, balanced speed
    - **gpt-3.5-turbo**: Faster, lower cost, less capable
  </Step>
  <Step title="Set Parameters">
    Adjust settings:
    - **Temperature**: 0.7 (default) for balanced creativity
    - **Max Tokens**: 1000 (default) for detailed responses
  </Step>
  <Step title="Test & Save">
    Test with sample query, then save configuration
  </Step>
</Steps>

### Fallback Mode

If OpenAI is not configured:

- AI Assistant still works with structured responses
- Uses pre-formatted templates
- No conversational depth
- Limited to predefined query types

<Note type="tip">
**Cost Estimate:** With gpt-4-turbo, typical query costs €0.02-0.05. For 100 queries/day, expect ~€3-5/day in API costs.
</Note>

## Advanced Features

### Semantic Customer Search

Find customers by description, not just name:

**Query:** "High-spending customers who love wine and haven't visited recently"

**AI Process:**
1. Identifies criteria: high spend + wine preference + inactive
2. Searches customer tags for "wine" or "wine lover"
3. Filters by spend (top 25%)
4. Filters by last visit (60+ days)
5. Returns ranked results

### SQL Generation (Advanced)

For complex queries, AI generates SQL:

**Query:** "Show me average party size by day of week for last month"

**AI Process:**
1. Recognizes need for grouping and aggregation
2. Generates SQL query (executed safely)
3. Formats results in readable table
4. Provides interpretation

<Note type="info">
SQL generation is sandboxed and read-only. No data modification possible.
</Note>

## Troubleshooting

<Accordion title="AI Not Responding">
**Possible Causes:**
- OpenAI API key invalid or expired
- Rate limit exceeded
- OpenAI service outage

**Solutions:**
1. Check Settings → Integrations → OpenAI
2. Verify API key is active in OpenAI dashboard
3. Check OpenAI status page
4. Falls back to structured mode automatically
</Accordion>

<Accordion title="Slow Responses">
**Causes:**
- GPT-4 models slower than 3.5
- Complex queries require more processing
- Large datasets (many customers/reservations)

**Solutions:**
- Switch to gpt-3.5-turbo for speed
- Reduce max_tokens to 500
- Be more specific in queries (narrows data set)
</Accordion>

<Accordion title="Inaccurate Answers">
**Causes:**
- Ambiguous question
- Missing context in data
- AI hallucination (rare)

**Solutions:**
- Rephrase question more specifically
- Ask for data source: "How did you calculate that?"
- Cross-reference with manual reports
- Report persistent issues to support
</Accordion>

## Technical Details

**Query Processing:**
- Average response time: 2-5 seconds (GPT-4)
- Timeout: 30 seconds
- Concurrent queries: Unlimited (rate-limited by OpenAI)

**Context Window:**
- Loads last 7 days of reservations
- Top 100 VIP customers
- Last 30 days of revenue data
- Ensures fast responses with relevant data

**Rate Limits:**
- Follows OpenAI tier limits (usually 10,000 requests/day)
- No Hiro-side limits
- Automatically retries on transient errors
